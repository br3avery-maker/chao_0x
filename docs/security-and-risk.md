# Security and Risk Notes

## Do not overclaim

Avoid claims like:

- bot-proof
- impossible to copy
- unkillable
- copyright-safe
- invisible daemon
- AI guarantees legality

Use:

- bot-resistant
- challengeable
- user-visible foreground service
- risk-scored
- provenance anchored
- index-only where needed

## Main risks

- copyrighted file monetization
- false creator claims
- malicious mini-app manifests
- metadata poisoning
- fake availability
- Sybil attacks
- bot-farmed taps
- stale indexes
- privacy leakage from local indexes
- battery/data abuse
- Android service restrictions

## Mitigations

- index-only mode
- risk scoring
- creator attestations
- stake/challenge/slash
- local encryption
- Merkle commitments
- rate limits
- permission sandbox
- explicit foreground service
- user approval for spends/signatures
